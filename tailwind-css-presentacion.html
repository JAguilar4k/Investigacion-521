Tailwind CSS como API de diseno compilada

Utility-first no elimina arquitectura: la mueve a tokens, variantes, componentes y un pipeline de generacion que produce CSS estatico, predecible y pequeno.

Evolucion arquitectonica

BEM/OOCSS vs CSS-in-JS vs Utility-first

Cada enfoque optimiza una dimension distinta: semantica global, colocacion por componente o generacion por uso real.

Dolores mitigados

El problema real no era escribir CSS: era mantenerlo

Bundle infinito

CSS tradicional tiende a crecer aunque el UI cambie. Las clases antiguas quedan vivas porque borrarlas puede romper pantallas no auditadas.

Tailwind genera utilidades detectadas en archivos fuente.

Especificidad rota

La cascada se vuelve una guerra entre selectores, orden de carga y excepciones. El siguiente cambio empieza con miedo.

Utilities mantienen selectores planos y composables.

Clases huerfanas

Nombres semanticos envejecen mal: la clase sigue diciendo hero-primary aunque ya no sea hero ni primary.

La semantica vive en componentes, no en nombres CSS globales.

Under the hood

Como piensa el motor JIT

Tailwind no ejecuta tu app ni parsea JSX como AST de negocio; detecta candidatos de clase en texto fuente y genera CSS para ellos.

// Fragil: Tailwind no ve clases completas
const klass = `text-${tone}-600`;

// Robusto: las clases existen como strings completos
const tones = {
  success: "text-emerald-600 bg-emerald-50",
  risk: "text-rose-600 bg-rose-50",
};

Configuracion avanzada

Extender, sobrescribir y nombrar tokens

v3: tailwind.config.js

export default {
  content: ["./src/**/*.{ts,tsx,html}"],
  theme: {
    extend: {
      colors: {
        brand: { 500: "#06b6d4" }
      }
    }
  }
};

v4: CSS-first

@theme {
  --color-brand-500: #06b6d4;
  --breakpoint-3xl: 120rem;
}

@custom-variant dark (&:where(.dark, .dark *));
Regla senior: usa `extend` para sumar a la escala; sobrescribe solo cuando el design system controla la escala completa.

Plugins

Plugins como extension del lenguaje visual

addUtilities

Clases estaticas, bajo riesgo.

matchUtilities

Valores dinamicos ligados al tema.

addComponents

Primitivos repetidos, no frameworks enteros.

matchVariant

Estados parametrizados por contexto.

plugin(({ matchUtilities, theme }) => {
  matchUtilities(
    { "scroll-mt-header": value => ({ scrollMarginTop: value }) },
    { values: theme("spacing") }
  );
});

Accesibilidad

Tailwind facilita estados accesibles, pero no reemplaza semantica

Variantes clave

  • focus-visible:ring-4
  • focus-within:*
  • motion-safe:*
  • aria-selected:*
  • data-[state=open]:*
  • peer-checked:*

Ejemplo de tab accesible

<button
  role="tab"
  aria-selected="true"
  class="focus-visible:ring-4
         aria-selected:bg-slate-950">
  Vision general
</button>

Patrones senior

La arquitectura vive en componentes, contratos y tokens

Hacer

  • Mapear variantes a strings completos.
  • Extraer componentes cuando hay intencion estable.
  • Nombrar tokens por dominio: brand, surface, risk.
  • Medir contraste y foco con teclado.

Evitar

  • Interpolar fragmentos de clases.
  • Convertir valores arbitrarios en norma.
  • Usar el CDN en produccion.
  • Crear plugins que reimplementan un framework completo.

Takeaways

  • Utility-first reduce CSS muerto y especificidad accidental.
  • JIT convierte clases usadas en CSS estatico.
  • v4 acelera builds con motor y configuracion CSS-first.
  • Accesibilidad requiere HTML semantico mas variantes de estado.

Notas: tesis

Abrir con la idea de API de diseno compilada. Tailwind no es ausencia de arquitectura; es arquitectura expresada en tokens y componentes.

Notas: comparativa

No vender Tailwind como bala de plata. Mostrar que cada enfoque paga costos distintos.

Notas: dolores

Conectar con experiencia real de equipos: miedo a borrar CSS, overrides crecientes y clases semanticas que ya no describen nada.

Notas: JIT

Enfatizar que las clases deben existir como strings completos. Esto explica muchos bugs de produccion.

Notas: performance

Los numeros son de benchmark oficial. El cambio relevante para DX es el incremental casi instantaneo.

Notas: bundle

Aclarar que las cifras de bundle son estimaciones. El tamano real depende de tokens, plugins y patrones de uso.

Notas: configuracion

Explicar `extend` contra sobrescritura y marcar el cambio de mentalidad en v4 hacia CSS-first.

Notas: plugins

Plugin solo cuando captura una regla compartida. Si requiere explicar mucho, probablemente es demasiado magico.

Notas: a11y

Tailwind ayuda a expresar estados, pero HTML semantico y contraste siguen siendo responsabilidad del equipo.

Notas: patrones

Esta slide es para convertir la charla en criterios de PR y convenciones de equipo.

Notas: demo

Usar el dashboard para aterrizar cada concepto: estado, responsive, dark mode, accesibilidad y clases arbitrarias.

Notas: cierre

Cerrar con la idea de gobierno: Tailwind escala si los componentes y tokens tienen duenos claros.
